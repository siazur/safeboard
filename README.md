# SafeBoard Cruise

Панель управления для круизного лайнера. Отслеживание пассажиров и членов экипажа.

## Ссылка на проект

GitHub: https://github.com/siazur/safeboard

![SafeBoard Preview](./src/img/safeboard.png)

## Бизнес-логика проекта

Приложение представляет собой внутреннюю систему для управления пассажирами и персоналом круизного лайнера. Все люди на борту делятся на две категории: пассажиры (группа "Гости") и члены экипажа (распределены по отделам: Капитанская служба, Кухня, Уборка, Техническая служба, Анимация, Медицина). Данные хранятся на сервере в формате JSON и загружаются при старте приложения. Реализована возможность просмотра, поиска, сортировки, добавления и удаления записей.

## Возможности

- Отображение всех на борту в виде таблицы
- Сортировка по клику на заголовки
- Поиск по имени и роли
- Добавление новых пассажиров и сотрудников, удаление с подтверждением
- Просмотр отделов и гостей на отдельной странице
- Адаптив под мобильные устройства, анимированный фон на главной странице.

## Технологии

- React
- React Router
- JSON Server
- CSS3 стили и адаптивность
- useMemo для оптимизации

## Установка и запуск

Клонировать репозиторий git clone https://github.com/siazur/safeboard.git, перейти в папку cd safeboard, установить зависимости npm install, запустить сервер с данными npm run server, запустить React приложение npm run dev, открыть http://localhost:5173 в браузере.

## Использование

- **Поиск** — введите текст в поле поиска и таблица отфильтруется по имени или роли
- **Сортировка** — нажмите на заголовок таблицы
- **Добавление** — нажмите на кнопку плюс справа от поиска, заполните форму и сохраните
- **Удаление** — нажмите на крестик в строке пользователя и подтвердите удаление

## Выводы по проектированию UI

**Ручное проектирование (страница Users)**  
При разработке страницы пользователей основное внимание уделялось созданию удобного интерфейса для работы с таблицей. Самостоятельно продумывалось расположение элементов, визуальные подсказки для сортировки (стрелочки у заголовков), цветовая схема в морских тонах, адаптивность под разные экраны. Основная сложность возникла с оптимизацией перерисовок при поиске и сортировке, но использование useMemo решило эту проблему. В результате получилась интуитивно понятная таблица, где все действия выполняются быстро и логично. Не был сразу учтен адаптив, он добавлялся уже в конце разработки страницы, так же как и сохранение данных.

**LLM-проектирование (страница Groups)**  
Для страницы групп был использован промпт с просьбой создать компонент с карточками отделов. LLM сразу выдала правильную структуру с карточками и списками сотрудников внутри. При этом важно учитывать, что нейросеть не знает заранее, какие цвета и стили используются в проекте — это нужно явно задавать в промпте. Например, пришлось дополнительно указывать использовать цвета #006F6E и #019493, чтобы привести страницу к общему стилю. Но если дать уже частично готовый проект с заданным дизайном, то нейросеть подхватит его. Зато LLM сразу учла адаптивность: в сгенерированном коде уже были медиа-запросы для мобильных устройств, что при ручной верстке часто забывают. Также структура компонента была логичной и не требовала переделок.
